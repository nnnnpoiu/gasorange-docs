# GasOrange 数据与隐私说明

## 导言

杭州橙之云科技有限公司（以下简称“GasOrange”“平台”或“我们”）重视用户的数据安全和个人信息权益。

本《数据与隐私说明》说明平台在用户访问网站、注册账户、充值付款、调用 API、使用控制台、提交工单或购买企业服务时，如何收集、使用、存储、共享、委托处理、传输、保护和删除数据，以及用户如何行使相关权利。

本说明应与《GasOrange 服务协议》、Cookie 说明、供应商清单、模型说明、企业订单和数据处理协议一并阅读。

平台只应按照合法、正当、必要、诚信和透明原则处理个人信息，并将处理范围限制在实现明确目的所必要的最小范围。

---

## 第一条 适用范围

本说明适用于：

1. GasOrange 网站及控制台；
2. 用户注册、登录、充值、计费和账户管理；
3. GasOrange API 和模型调用；
4. 企业接入、合同、发票、对账和技术支持；
5. 工单、邮件、在线客服、社群和安全沟通；
6. 平台明确引用本说明的其他服务。

本说明不直接适用于：

1. 用户通过自身产品或服务收集和处理的终端用户数据；
2. 第三方独立运营的网站、应用、支付工具或模型服务；
3. 企业用户自行决定的数据处理活动。

第三方服务适用其自身隐私规则。平台将尽合理努力选择符合安全和合规要求的服务商，但用户也应查阅相关第三方规则。

## 第二条 平台在不同场景中的角色

### 2.1 账户、支付和平台运营数据

对于用户注册信息、账户管理、充值、账单、安全风控、客服沟通、网站分析和平台运营数据，平台通常作为个人信息处理者，自主决定处理目的和方式。

### 2.2 企业用户通过 API 提交的数据

企业用户通过 API 提交其客户、员工、用户或其他第三方的个人信息时：

1. 企业用户通常决定处理目的和方式；
2. 平台通常按照企业用户指示提供传输、路由、调用、返回、计费和技术支持；
3. 双方可另行签署数据处理协议，明确委托处理的目的、期限、方式、数据类型、安全措施、转委托、删除和审计安排；
4. 如果平台超出企业用户指示，自主决定新的处理目的，平台可能就该部分承担相应的个人信息处理者责任。

### 2.3 用户责任

企业用户应确保：

1. 对 API 内容中的个人信息拥有合法处理基础；
2. 已向相关个人履行必要告知；
3. 已取得依法需要的同意、单独同意或书面同意；
4. 不向平台提交超出业务必要范围的信息；
5. 对敏感个人信息、未成年人信息、重要数据和受监管数据进行事前评估；
6. 在需要时与平台签署专项数据处理协议。

- 企业客户在人工沟通、签约、开票或合规审查中提供的企业名称、统一社会信用代码、联系人、职位、联系方式、开票资料及必要的企业证明材料。

处理目的：

- 创建和管理账户；
- 身份验证和找回账户；
- 通过官方客服进行企业需求确认、签约、开票和必要的主体核验；

- 防止欺诈和滥用。

- 充值金额、支付时间和支付状态；

- 拒付和交易争议记录。

处理目的：

- 完成支付和充值；
- 计费和对账；
- 开具发票；
- 财务审计、税务和反欺诈；

### 3.3 服务使用和技术日志

- 安全检测和滥用防范；
- 生成账单、统计和用量报告；
- 履行审计和监管要求。

### 3.4 API 内容

可能包括：

- 提示词和系统消息；
- 对话上下文；

## 第五条 API 内容的处理规则

### 5.1 必要处理

为完成模型调用，平台可能对 API 内容进行：

- 接收；
- 格式转换；
- 加密传输；
- 路由至上游模型服务商；
- 临时缓存；
- 内容安全检测；
- 故障排查；
- 返回模型输出；
- 按规则删除或匿名化。

### 5.2 内容日志与保存

1. GasOrange 使用中转系统自动接收、转发和处理用户输入、模型输出及调用参数，以完成身份鉴权、路由、模型调用、流式传输、异步任务、结果返回、计费、安全防护和故障排查。
2. 平台原则上不将完整提示词、对话上下文和模型输出作为长期业务档案保存。因请求转发、任务状态管理、错误排查、安全检测、内容治理或用户主动申请技术支持而产生的临时内容记录，默认保存不超过 **7 日**。
3. 对图片、音频、视频、文件和异步任务，平台或上游服务商可能在生成、传输、下载和任务状态查询所必要的期限内进行临时存储。GasOrange 自有系统中的相关临时文件默认保存不超过 **7 日**；模型页面或双方书面约定另有说明的，从其说明。
4. API 调用元数据，包括调用时间、模型名称、请求标识、任务标识、Token 或任务用量、响应时间、请求状态、错误码、IP 地址、消费金额及相关计费信息，默认保存 **180 日**。
5. 登录、操作、访问控制、风险识别和网络安全日志默认保存不少于 **180 日**。
6. 用户通过客服微信或联系邮箱主动提供的提示词、截图、请求样例、文件和日志，仅用于解决其提出的问题、安全调查或争议处理，默认保存至问题关闭后 **180 日**。
7. 数据库、日志和配置备份采用滚动覆盖方式，默认保存不超过 **30 日**。相关数据从主系统删除后，可能在备份覆盖周期内继续存在；除灾难恢复、安全调查或法律要求外，平台不会从备份中单独恢复已删除数据。
8. 因履行财税、会计、审计、合同、争议处理、司法、行政或监管义务需要保存的账户、交易、发票、合同和相关记录，按照适用法律要求或实现上述目的所必要的期限保存。
9. 如发生欺诈、滥用、安全事件、拒付、计费争议、侵权投诉或违法违规调查，平台可在完成调查和法定责任处理所必要的范围内延长相关记录的保存期限。
10. 企业用户可通过企业订单或数据处理协议申请更短的内容保存期限、无内容日志或其他专项安排。是否提供、适用范围和技术例外，以双方书面确认和平台实际技术能力为准。

### 5.3 模型训练和产品改进

1. 除非取得用户明确授权或企业订单另有约定，GasOrange 不使用用户 API 内容训练平台自有的通用人工智能模型。
2. 平台可使用不包含可识别 API 内容的聚合、统计或去标识化服务数据，用于改进路由、计费、性能、安全、风控和产品体验。
3. 用户主动请求故障排查时，平台工作人员可在必要范围内查看用户自行提供或系统记录的相关请求信息；该处理仅限于解决问题、安全调查或履行法定义务。
4. 不同上游模型服务商可能适用不同的保留和训练政策。企业客户可要求指定不使用其 API 内容训练模型的渠道，但是否可提供、适用模型、价格和其他条件，应在接入前通过书面文件确认。

### 5.4 上游模型服务商

1. 为完成请求，API 内容可能被传输给用户所选模型对应的上游服务商。当前可能使用的服务商或渠道包括 AWS Bedrock、Microsoft Azure 以及其他模型聚合服务商。
2. 不同上游服务商可能具有不同的数据区域、保留、训练、安全和内容审核政策。
3. 平台应维护并公开或向企业客户提供供应商清单，至少说明：
   - 服务商名称或类别；
   - 提供的服务；
   - 可能处理的数据类型；
   - 数据处理地区；
   - 是否可能跨境；
   - 适用的保留和训练政策；
   - 变更通知方式。
4. 若企业客户要求固定供应商、固定地区或不用于训练，应在接入前书面确认并写入企业订单或数据处理协议。

## 第六条 敏感个人信息与受限制数据

1. 敏感个人信息一旦泄露或非法使用，可能对个人权益造成较高风险。
2. 用户不应通过公共或普通 API 渠道提交以下数据，除非平台明确支持、双方完成评估并采取增强保护措施：
   - 生物识别信息；
   - 宗教信仰；
   - 特定身份；
   - 医疗健康信息；
   - 金融账户和信用信息；
   - 精确行踪轨迹；
   - 不满十四周岁未成年人的个人信息；
   - 国家秘密、工作秘密；
   - 已被认定或可能构成重要数据的数据；
   - 法律法规或监管规则严格限制的数据。
3. 如业务确需处理，企业用户应提前联系平台，明确：
   - 处理目的和必要性；
   - 数据类型和规模；
   - 合法处理基础；
   - 存储和跨境要求；
   - 加密、访问、日志和删除措施；
   - 事件响应和责任安排。
4. 平台有权拒绝无法满足安全和合规要求的处理请求。

## 第八条 数据存储与跨境传输

### 8.1 基础设施与 IP 地址说明

1. GasOrange 使用阿里云及其他必要的云计算、网络和安全基础设施提供服务。
2. 部分对外服务 IP 地址、网络出口或 IP 地理位置数据库可能显示为美国或其他境外地区。**IP 地址的注册地、广播位置或第三方地理识别结果，不必然等同于服务器物理位置、数据库存储位置或数据实际处理地点。**
3. 平台不会仅根据公开 IP 地址的地理显示结果，将用户数据描述为存储在美国。账户、账单、服务元数据、日志、文件和备份的实际处理位置，以平台部署的云资源区域、网络架构及上游服务商实际处理路径为准。
4. 出于安全和供应链管理考虑，平台可以不在公开页面披露具体服务器地址、IP、机房或内部网络拓扑，但会按照适用法律说明可能的数据跨境处理情况，并可根据企业客户的合理合规需求提供必要说明。

### 8.2 模型调用与可能的跨境处理

1. 平台可能使用 AWS Bedrock、Microsoft Azure 及其他模型聚合服务商提供模型能力。
2. 根据用户选择的模型、实际路由渠道、云服务区域和上游架构，API 内容、调用元数据、异步任务数据、安全日志或技术支持资料可能被传输至中国大陆以外的地区并由相关服务商处理。
3. 不同上游服务商的数据保留、训练、访问控制、内容审核和处理地区可能不同。GasOrange 不仅根据公开 IP 地址判断数据是否跨境，而是根据实际服务链路和供应商处理安排进行判断。
4. 企业用户对固定上游、固定数据区域、不用于训练、无内容日志或不跨境有要求的，应在测试和正式采购前提出。平台将根据实际供应链和技术能力评估，并以企业订单、数据处理协议或其他书面文件为准。
5. 在未作书面确认前，平台不承诺用户数据仅在中国大陆境内处理、绝不跨境或固定在某一特定国家或地区。

### 8.3 跨境保护安排

如实际业务涉及个人信息出境或其他依法受监管的数据跨境活动，平台将根据数据类型、处理规模、业务必要性和适用法律，采取相应措施，包括：

- 对跨境处理的合法性、正当性、必要性和风险进行评估；
- 按照适用要求履行告知义务；
- 在依法需要时取得单独同意；
- 与境外接收方或相关服务商约定数据保护义务；
- 在适用时履行标准合同、认证、安全评估、备案或其他程序；
- 采取数据最小化、传输加密、访问控制、日志审计和供应商管理措施。

### 8.4 企业客户选择

企业客户如对数据处理地区或供应链有明确要求，应在接入前联系官方客服。平台可根据模型、渠道、调用规模和服务方案评估固定区域、固定渠道、专属资源或其他数据保护安排，最终以双方书面确认内容为准。

## 第九条 数据保存期限

我们仅在实现处理目的所必要的最短期限内保存个人信息，法律法规另有要求的除外。

默认保存规则如下：

| 数据类别 | 默认保存期限或规则 |
|---|---|
| 账户基本信息 | 账户存续期间；账户注销后，为履行争议处理、安全和法定义务所必要的期限内保存，之后删除或匿名化 |
| 企业合同、交易、账单和发票资料 | 按适用的财税、会计、审计、合同和监管要求保存 |
| API 完整请求和输出 | 原则上不作长期保存；因服务运行、错误排查、安全或客服支持产生的临时记录不超过 7 日 |
| 图片、音频、视频和文件临时数据 | GasOrange 自有系统默认不超过 7 日；上游规则或书面约定另有规定的除外 |
| API 调用元数据 | 180 日 |
| 登录、操作和网络安全日志 | 不少于 180 日 |
| 数据库、日志和配置备份 | 滚动保存不超过 30 日 |
| 客服沟通和排障资料 | 问题关闭后 180 日 |
| 营销联系信息 | 保存至用户退订、撤回同意或达到处理目的 |
| 安全事件、计费争议和投诉资料 | 保存至事件或争议处理完成，并满足适用的追责和法定义务期限 |

确定具体期限时，我们会考虑：

- 提供服务和履行合同所需时间；
- 数据类型、敏感程度和风险；
- 计费、对账和争议周期；
- 安全、防欺诈和审计需求；
- 法律、监管、司法和行政要求；
- 企业订单的特别约定。

如果删除在技术上暂时难以实现，我们将停止除存储和安全保护之外的处理。

12. 数据分类分级、去标识化或匿名化；
